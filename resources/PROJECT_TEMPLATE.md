> *This section is only for Capstone Projects. Delete for mini projects.*

### What Makes a Capstone Different?

A capstone project is your **showcase** — it combines everything from the entire track into one meaningful, complete project. It should represent your best work.

### Requirements

| Category | Requirement |
|---|---|
| **Functionality** | [Core features that must work] |
| **Code Quality** | Comments, clean structure, meaningful variable names |
| **Complexity** | Must use at least [N] concepts from the track |
| **Documentation** | Full README.md with screenshots/demo description |
| **Creativity** | Something personal — make it your own |

### Presentation Guide

When sharing your capstone:
1. **Demo first** — show it working before explaining the code
2. **Explain one interesting part** — walk through the trickiest piece
3. **Share what you learned** — what was hard? what surprised you?
4. **What would you add next?** — show you're still thinking
